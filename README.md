Цей проєкт є курсовою роботою з дисципліни **"Архітектура та проєктування ПЗ"**. Це модель високопродуктивного кеш-сервера, який демонструє практичне застосування 10 різних патернів проєктування для вирішення завдань кешування, безпеки та аналітики.

## 🛠 Технологічний стек
* **Мова:** C++17
* **Середовище:** Visual Studio 2022
* **Контейнери:** `std::unordered_map`, `std::vector`, `std::queue`
* **Керування пам'яттю:** Розумні вказівники (`std::shared_ptr`, `std::dynamic_pointer_cast`)

## 🧩 Реалізовані патерни проєктування

Проєкт побудований за принципами SOLID і включає наступні патерни GoF:

### 🏗 Твірні патерни (Creational)
* **Factory Method:** Використовується для створення об'єктів кешу та сервісів.
* **Builder:** Покрокова генерація складних відповідей сервера (`ResponseBuilder`).

### 🏛 Структурні патерни (Structural)
* **Facade:** Надає спрощений інтерфейс для керування всією складною системою сервера.
* **Proxy:** Реалізує механізм кешування та контролю доступу перед зверненням до бази даних.
* **Decorator:** Дозволяє динамічно додавати функціонал.
* **Flyweight:** Оптимізує використання ОЗП через спільне використання метаданих сервера.

### ⚙️ Поведінкові патерни (Behavioral)
* **Strategy:** Дозволяє на льоту змінювати алгоритми витіснення даних (**LRU** або **FIFO**).
* **Observer:** Система подій для логування звернень до кешу (HIT/MISS).
* **Chain of Responsibility:** Ланцюжок фільтрів для перевірки безпеки та прав доступу.
* **Visitor:** Використовується для збору статистики та аналізу стану об'єктів у пам'яті.

## 🚀 Основні можливості
1. **Динамічне керування стратегіями:** Можливість перемикання між алгоритмами витіснення без зупинки сервера.
2. **Багаторівнева безпека:** Перевірка прав адміністратора та фільтрація заборонених ключів.
3. **Аналітика пам'яті:** Детальний звіт про обсяг даних та популярність об'єктів.

## 📂 Структура проєкту
Кожен модуль патерна винесений у окрему папку для кращої навігації та дотримання принципу Single Responsibility.
